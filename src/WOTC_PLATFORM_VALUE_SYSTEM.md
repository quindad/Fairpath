## ðŸš€ Implementation

### **Database Fields:**
```sql
users
  - wotc_value (calculated: 2400 or 5600-9600)
  - wotc_status ('pipeline', 'available', 'claimed')
  - fpf_to_platform_conversion_date
  - hire_date (when claimed)

platform_wotc_totals
  - total_value
  - pipeline_value
  - available_value
  - claimed_month_value
  - updated_at
```

### **Calculation Logic:**
```javascript
// On FPF signup
user.wotc_value = platform === 'faf' ? 2400 : 9600;
user.wotc_status = 'pipeline';
platform_total += user.wotc_value;

// On release/separation
user.wotc_status = 'available';
// Total stays same, just moves from pipeline to available

// On hire
user.wotc_status = 'claimed';
user.hire_date = NOW();
platform_total -= user.wotc_value;
claimed_month_value += user.wotc_value;
```

### **Real-Time Updates:**
```javascript
// Supabase realtime subscription
supabase
  .from('users')
  .on('INSERT', (payload) => {
    if (payload.new.fpf_profile) {
      platformValue += payload.new.wotc_value;
      showNotification(`+$${payload.new.wotc_value} pipeline`);
    }
  })
  .on('UPDATE', (payload) => {
    if (payload.new.wotc_status === 'claimed') {
      platformValue -= payload.new.wotc_value;
      showNotification(`-$${payload.new.wotc_value} claimed`);
    }
  })
  .subscribe();
```
